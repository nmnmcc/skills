# Example Skill

Follow these steps:

1. Identify the user's requested outcome and the files or systems in scope.
2. Inspect relevant context before making changes.
3. Perform the smallest complete set of actions needed to achieve the outcome.
4. Validate the result with an appropriate check.
5. Report the outcome, changed artifacts, and any remaining caveats.

Replace these generic instructions with domain-specific knowledge that is not obvious to a capable agent. Remove unused resource directories when customizing this example.

## Resources

- Run deterministic or frequently repeated operations from `scripts/`.
- Read detailed supporting documentation from `references/` only when relevant.
- Reuse templates and static output files from `assets/`.
